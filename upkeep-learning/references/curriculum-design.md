Treat curriculum design as a first-class responsibility.

## Core Rule

Build a network of study routes, not a single rigid syllabus.

The archive should support:

- bottom-up learning,
- top-down project understanding,
- implementation-first study,
- subsystem-focused study,
- domain-theory-first study,
- roadmap/future-direction study,
- interview or explanation fluency when relevant.

## Route Design

Every meaningful learning path should answer:

- who this path is for,
- what it assumes,
- what order to follow,
- what the learner should understand by the end,
- which related paths or files to do next.

Good paths should include:

- concepts,
- project files,
- exercises,
- glossary support where needed,
- materials for extra depth.

## Prerequisite Flow

Do not assume the learner already knows upstream theory.

If a major project file depends on difficult theory:

- create concept files for that theory,
- cross-link them explicitly,
- make paths route through them when needed.

## Layered Depth

The archive should let learners move through different depth layers:

1. orientation and route selection,
2. foundational concepts,
3. project-specific understanding,
4. advanced theory and trade-offs,
5. exercises and reconstruction,
6. historical comparisons and future direction.

The archive is strongest when these layers reinforce each other.

## Educational Momentum

Paths and guide files should create momentum.

Use:

- clear route selection,
- explicit sequencing,
- milestone framing,
- meaningful combinations,
- “what to do next” sections,
- project-grounded outcomes.

Do not leave the learner with isolated documents and no sense of progression.

## Large-Archive Strategy

In a large archive, create multiple paths instead of stuffing all progression into:

- one giant `CURRICULUM.md`,
- one giant `STUDY_GUIDE.md`,
- one giant `PATH_INDEX.md`.

High-level route files should guide the learner into deep topic files and subpaths.

## Path Diversity

Useful path types often include:

- `foundations-path.md`
- `project-architecture-path.md`
- `implementation-first-path.md`
- `domain-theory-path.md`
- `advanced-topics-path.md`
- `comparative-analysis-path.md`
- `debugging-and-observability-path.md`

Create the paths the repository actually warrants.

## Curriculum Failure Modes

Treat these as failures:

- only one path through the archive,
- paths that are just unordered link dumps,
- paths that ignore prerequisites,
- paths with no exercises or no project grounding,
- paths that ignore major README-defined learning surfaces,
- navigation files that are too thin to guide a large archive.
